# 目录命名规律 / 管理面暴露链路

## 沉淀边界

- 必须保留：高风险停止条件、凭据/云资源/生产运维面的红线。
- 必要流程：目标命名规律 -> 有界词表 -> 管理面只读识别 -> 访问记录/配置字段反哺二次 recon。
- 可复用经验：目标特定词表通常优于通用目录字典；管理面只读 JSON/访问记录常能暴露更高价值路径、参数和 secret 候选。
- 不沉淀为 autopilot 默认动作：口令爆破、批量默认凭据尝试、导入云管平台、接管服务器、读取真实数据或触发支付/运维动作。口令测试可作为受控 `/spray` / `credential-attack` 后续动作记录。

## 适用场景

- 目标 URL、子目录、子系统名存在明显命名规律，例如 `wq1`、`ax1`、`app01`、`web-a`、`admin-v2`。
- 常规目录字典效果一般，但目标自身路径、JS、历史 URL、访问日志或管理面记录暴露了更贴近业务的词。
- 发现 Druid、Actuator、Spring Boot Admin、Grafana、Kibana、Nacos、Consul、Jenkins、监控后台、日志后台、数据库连接池面板等管理/监控入口。
- 只读接口、访问记录、配置页、健康检查、weburi/url 统计或日志索引可能泄露内部路径、接口、配置字段或 access key。

## 触发信号

- 某个目录命中后，兄弟目录像是短字符串、业务缩写、环境编号、部门代码或版本号组合。
- 管理面返回 200/301/302/401/403，标题、路径或静态资源显示 Druid、monitor、admin、console、actuator、metrics、weburi、datasource、log 等关键词。
- 管理面登录页使用默认品牌、缺少验证码/锁定提示、错误信息稳定；这可以形成受控口令测试 lead，但不得由本卡或 `/autopilot` 自动进入爆破。
- 只读 JSON、访问记录、配置字段或日志里出现 `accesskey`、`secretKey`、`ak`、`sk`、`token`、`bucket`、`endpoint`、`region`、内部 API 路径。

## 发散问题

- 当前路径命名是不是能生成更小的目标特定字典，而不是直接跑通用大字典？
- 命中的兄弟目录是否共享同一套部署、静态资源、登录方式、API 前缀或管理面？
- 管理面是否无需认证、弱认证、默认凭据、只读可访问，或暴露访问记录/配置字段？
- 访问记录里的长 URL、JSON 接口、内部路径是否能反向构造新的 recon 字典？
- 疑似云密钥或 access key 是否能在不接管资源、不读取真实数据的前提下做最小有效性确认？

## 推荐动作

- 先从目标自身提取词表：路径分段、目录前缀/后缀、短码模式、JS 路由、历史 URL、管理面访问记录。
- 对命名规律做有界生成：限制长度、字符集和候选数量，优先验证最像目标风格的兄弟目录。
- 管理面只先做安全识别：标题、版本、认证要求、只读页面、公开 JSON、配置字段；口令测试作为手动 `/spray` 或 `credential-attack` 后续动作。
- 发现访问记录或 weburi/json 类接口时，优先提取路径和参数作为二次 recon 字典，不直接访问状态改变接口。
- 发现 access key/secret/token 时，只记录来源、字段名、上下文和最小有效性验证计划；需要云资源枚举、导入第三方工具或接管资源时先停为 Candidate/blocked。

## 停止条件

- 目录规律产生的候选数量过大，已经接近无边界爆破或触发限速/WAF/告警风险。
- 管理面需要口令爆破、默认凭据检查或真实账号尝试时，当前管理面暴露 lane 停止并转手动 `/spray` / `credential-attack`；验证码绕过、账号锁定规避或大规模猜测按红线处理。
- 访问记录里的 URL 指向写入、删除、支付、订单、消息发送、部署或其他状态改变动作。
- 疑似云密钥验证需要列举真实资源、读取客户数据、导入云管平台、接管服务器或执行运维动作。

## 检查要求

- 不把“存在登录页”直接当作漏洞；Candidate 需要可达性、认证缺陷、敏感信息泄露或配置影响证据。
- 默认凭据检查必须低频、有限、可说明依据；如需扩大为口令爆破，必须切换到手动 `/spray` / `credential-attack` 受控流程。
- Secret 候选只保留最小证据，避免保存完整密钥、批量导出资源或扩大影响。
- 任何云、支付、生产服务器、CI/CD、部署和运维面操作必须先按 `rules/red-lines.md` 降级或停止。

## 可晋升经验

- 某类目录短码、业务前缀或环境编号能稳定生成高命中目标词表。
- 某类管理面只读接口经常泄露二次 recon 字典或 secret 字段。
- 某类访问记录字段能把“目录 fuzz”转化为高价值 API、配置和 secret 检查路线。
