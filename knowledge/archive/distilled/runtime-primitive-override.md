# 同 realm 内覆盖原语/内建方法击穿安全控制

## 适用场景

- 安全校验函数与不可信代码同处一个 realm/作用域
- 存在 Electron preload、扩展内容脚本、共享全局的桥
- 后端存在可被污染的原型/数据源

## 触发信号

- 校验依赖可被覆写的内建/全局/原型方法
- preload/桥在同作用域暴露特权 API
- 数据源可注入 __proto__/构造器达 sourceURL/RCE

## 发散问题

- 校验和被保护逻辑是否共享同一可变全局？
- 攻击者代码能否先运行并覆写校验原语？
- 原型污染能否流到执行 sink？

## 推荐动作

- 定位校验依赖的可变原语，尝试覆写观察绕过。
- 对桥/preload 测同作用域 monkeypatch。
- 追踪原型污染到具体 sink。

## 关联 Skills

- web2-vuln-classes
- mobile-pentest
- triage-validation

## 停止条件

- 校验运行在不可变/隔离 realm，原型冻结
- 污染无法到达任何 sink

## 检查要求

- 必须证明覆写/污染实际绕过了安全控制或达成执行，且可复现。

## 可晋升经验

- "校验和攻击者代码同 realm"即高危；先找可覆写的校验原语。
