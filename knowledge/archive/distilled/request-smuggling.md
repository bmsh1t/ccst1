# HTTP 请求走私 / 降级 / 伪首部注入

## 适用场景

- 链路存在多级代理/网关/CDN 与后端
- 支持 HTTP/2 且会降级到 HTTP/1.1 转发
- 前后端可能用不同库解析请求边界

## 触发信号

- HTTP/2 伪首部允许注入空白/控制字符
- header 名/值转换在降级时产生歧义
- Content-Length / Transfer-Encoding 处理不一致

## 发散问题

- 前端和后端在哪里对请求边界产生分歧？
- H2->H1 降级时伪首部如何被重写？
- 能否让一个请求被后端解析成两个？

## 推荐动作

- 低频、单请求地探测边界解析差异，避免高压流量。
- 用可观测但无破坏的探针确认走智，而非批量注入。
- 记录前后端各自的解析结果。

## 关联 Skills

- web2-vuln-classes
- security-arsenal
- triage-validation

## 停止条件

- 前后端解析一致或统一由单点终止
- 只能靠高压/大量请求才有信号（违反红线）

## 检查要求

- 必须证明可控地让后端把一个连接上的字节切分成攻击者预期的请求，且低风险可复现。

## 可晋升经验

- desync 关注"边界解析分歧点"而非流量强度；红线禁止压力式验证。
