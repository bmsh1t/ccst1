# Advanced Mode Flags — Parallel Workers + Phase 4 Calibration

Moved out of `commands/autopilot.md` to keep the autopilot prompt lean. The
flags below are rarely needed; load this page only when one of them is
explicitly requested.

## Parallel Sibling Workers

Default is single-thread. To fan out sibling-probe execution after a primary
finding is confirmed, opt in with `--parallel`.

| Flag | Purpose | Default |
|---|---|---|
| `--parallel` | Enable B6 worker fanout when a primary finding emits a sibling-probe queue | off |
| `--max-parallel N` | Cap on concurrent workers; paranoid forces 1, yolo allows up to 8 | 3 |
| `--worker-timeout-secs N` | Per-worker wall-clock timeout | 300 |

Each worker writes only to `evidence/<target>/workers/sibling-<id>/`
(`attempts.jsonl`, `findings.json`, `done.flag`). The parent agent joins on
`done.flag`, dedups by `(endpoint, vuln_class)`, appends consolidated rows to
`findings/<target>/findings.json`, and re-runs `coverage_matrix.rebuild` so
the F3 gate sees the new tested_clean cells.

Audit-log rows from a worker carry additive `worker_id` and `parent_session`
fields; existing rows without them remain valid. The shared rate-limiter
state lives at `hunt-memory/audit/parallel_lock.json` and is coordinated via
`fcntl.flock`.

## Phase 4 Calibration & Self-Review Flags

| Flag | Purpose | Default |
|---|---|---|
| `--calibrate-patterns` | (B12d) Pass `calibrated=True` to `PatternDB.match()`; deprioritise patterns with samples ≥ 5 and precision < 0.2 | off |
| `--self-review` | (B12c) Spawn an adversarial red-team review worker before /report on /validate PASS | off |
| `--parallel-hypotheses` | (B12a) Fan out viable working_hypothesis candidates to workers in parallel | off |
| `--vision` | (B12b) Capture playwright screenshots and expose `read_browser_screenshot` to vision-capable models | off |
| `--max-screenshots N` | (B12b) Cap screenshots per page navigation | 5 |

Calibration data lives at `hunt-memory/pattern_calibration.jsonl` (size-based
rotation, 10MB cap, 3 backups). The `pattern_calibration_summary` dispatcher
tool surfaces the per-pattern precision/recall table mid-hunt.

After P5-W1, the Phase 4 primitives are also reachable from the LLM through
these dispatcher tools (gated by the same flags):

| Tool | Required flag | Primitive |
|---|---|---|
| `run_sibling_probe(seed_findings)` | `--parallel` to fan out, else sequential | `spawn_sibling_worker` (B6) |
| `run_hypothesis_fleet(hypotheses)` | `--parallel-hypotheses` to fan out, else sequential | `run_fanout` (B12a) |
| `run_vision_probe(url)` | `--vision` AND vision-capable model | `capture_with_screenshot_sequence` (B12b) |
| `run_self_review(candidates)` | `--self-review` (otherwise no-op) | `spawn_red_team_worker` + `decision_for` (B12c) |
| `record_validation_calibration` (auto) | always-on inside `/validate` | `pattern_calibration.record_outcome` (B12d) |

`--paranoid` mode forces `max_parallel=1` regardless of the requested value
(via `coerce_max_parallel`). `--normal` caps it at 3 and `--yolo` at 8.
