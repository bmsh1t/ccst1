# v4.5.1 — Hypothesis-Transition Journal Roadmap (Item 4, Option 4A)

## Status

**Deferred** to docs-only in v4.5.1. Functional path captured here for v4.6
or later. Approved by user as Option 4A on the v4.5.1 planning round.

## Problem (recap from v4.5.0 results)

The v4.5 `Mandatory Reflection Cadence` block already requires the LLM to
emit a one-line `[H] hypothesis alive? [N] next_question? [P] sibling/bypass/chain?`
after every tool result. v4.5.0 lab runs showed the cadence does fire in
chat text — but the resulting reflection is **opaque to downstream tooling**:

- `hunt-memory/journal.jsonl` records tool calls, not hypothesis transitions.
- `tools/coverage_matrix.py` and `agent.py` cannot read what the LLM was
  pursuing at the moment it pivoted to a new lane.
- A `/pickup` summary cannot reconstruct "which hypothesis was alive when
  the previous run ended", only "which tool fired last".

End result: rich human-readable reflection, zero structured signal for
later automation.

## Why deferred in v4.5.1

The user-standing constraint **"尽量不要做重构工作,侧重点在渗透测试上"**
ruled out the v4.5.0 plan's Option 4B (touch `agent.py` to wrap reflection
into a typed event before tool dispatch). That option would force every
agent turn through a new instrumentation layer and re-test the full
autopilot session lifecycle.

Option 4A keeps the functional gain on the v4.6 backlog while v4.5.1
spends its budget on Items 3 + 5 (cadence labs + class fingerprints) that
deliver immediate hunting-quality lift.

## Recommended path (Option B, scheduled for v4.6)

### Surface

Extend `memory/hunt_journal.py` with a new event type
`hypothesis_transition`. No `agent.py` change required. The LLM is taught
(via prompt edit) to call the new helper from inside its existing tool-
calling loop whenever it would emit a `[H][N][P]` line:

```python
# memory/hunt_journal.py (sketch)
def record_hypothesis_transition(
    target: str,
    hypothesis: str,         # current hypothesis under test
    status: str,             # "alive" | "stalled" | "killed" | "promoted"
    next_question: str,
    pivot: str | None = None # sibling / bypass / chain hint
) -> None:
    _append_jsonl(
        path=_journal_path(target),
        row={
            "ts": _utc_now(),
            "event_type": "hypothesis_transition",
            "target": target,
            "hypothesis": hypothesis,
            "status": status,
            "next_question": next_question,
            "pivot": pivot,
        },
    )
```

### Schema

Add to `memory/schemas.py`:

```python
HypothesisTransition = TypedDict(
    "HypothesisTransition",
    {
        "ts": str,
        "event_type": Literal["hypothesis_transition"],
        "target": str,
        "hypothesis": str,
        "status": Literal["alive", "stalled", "killed", "promoted"],
        "next_question": str,
        "pivot": str | None,
    },
    total=True,
)
```

`memory/rotation.py` and `memory/audit_log.py` handle the new event type
transparently — they iterate JSONL rows and don't care about the schema
beyond `ts` + `event_type`.

### Prompt edit in autopilot.md

```markdown
### After every tool result

Write one line before the next tool call:

    [H] hypothesis alive? [N] next_question? [P] sibling/bypass/chain?

Then, when running under a target with a writable journal, persist the
same transition via `memory.hunt_journal.record_hypothesis_transition`.
The plaintext line stays in chat for human readers; the structured row
lets /pickup, coverage_matrix, and future analytics see the hypothesis
graph instead of just the tool call graph.
```

### CLI surface (read side)

Add a one-shot `tools/hypothesis_timeline.py --target <t>` that prints
the ordered `hypothesis_transition` events for a target. /pickup can
shell out to it.

### Acceptance

- `pytest tests/test_hunt_journal.py::test_hypothesis_transition_roundtrip`
- `/pickup <t>` shows last 5 transitions instead of just last 5 tool calls
- `coverage_matrix.py find-gaps` can read transitions to deprioritize
  cells the LLM explicitly killed (not just untested)

### Cost estimate

- `memory/hunt_journal.py`: +30 LOC
- `memory/schemas.py`: +12 LOC
- `tools/hypothesis_timeline.py`: ~50 LOC
- `commands/autopilot.md`: +5 lines in the reflection block
- 1 new pytest: ~30 LOC
- Total: ~125 LOC + 1 prompt edit, all in `memory/` + `tools/`
- **No `agent.py` touch**, preserving the "尽量不重构" constraint

### Risk

Lowest-risk extension path that adds structure to the cadence. JSONL
append is already used by every other journal event. The LLM either emits
the row or it doesn't — failing to emit doesn't break the run, only
loses one timeline entry.

## What v4.5.1 actually ships for Item 4

- This roadmap document (`docs/v4.5.1-journal-hypothesis-roadmap.md`)
- No production code change
- Existing `[H][N][P]` chat-text reflection remains the source of truth
  until Option B lands

## When to revisit

Trigger conditions for picking this up:

1. A `/pickup` user complains the summary lost context across hunts on
   the same target
2. `coverage_matrix.py find-gaps` shows the same cells re-tested across
   runs because the prior kill is invisible
3. A v4.6 milestone reserves agent-internal budget for the `memory/`
   extension (orthogonal to autopilot prompt work)

Until any of those fire, the cadence text path is sufficient.
