## Item-5 design pivot (mid-v4.5.1)

An earlier v4.5.1 draft shipped `tools/fingerprint_match.py` + 8 class
fingerprint files under `hunt-memory/fingerprint_classes/`. User review
flagged the design as over-engineered:

> 8 files, ~350 lines, ~80% of the content is training-data knowledge the
> LLM already has (OAuth `redirect_uri` bypass, SAML XSW, GraphQL
> introspection, etc.). `autopilot.md` already names per-stack 3-bypass
> payloads in the Cadence block. Splitting into 8 files + a regex selector
> adds maintenance load without giving the LLM signal it didn't already
> have.

**Resolution** — delete `tools/fingerprint_match.py` and the
`hunt-memory/fingerprint_classes/` directory. Replace with a 12-line
**Step 0.6: Stack Recall** block in `commands/autopilot.md` that forces
the LLM to name the stack from recon evidence and explicitly recall the
relevant bug families before HUNT.

The LLM already knows OAuth/SAML/GraphQL/IIS attack patterns. The only
real gap was: it sometimes opens HUNT with a generic IDOR sweep instead
of dispatching the stack-specific class. A prompt nudge fixes that
without 350 lines of redundant markdown.

## What changed

### Item 5 — Step 0.6 stack recall (autopilot.md only)

```diff
@@ commands/autopilot.md @@
+## Step 0.6: Stack Recall
+
+After Step 0.5, name the stack from recon evidence (`Server` /
+`X-Powered-By` headers, JS bundle imports, URL shape, framework markers
+like `__NEXT_DATA__` / `app-root`) and explicitly recall the relevant bug
+families from training data before HUNT — e.g. OAuth `redirect_uri` /
+state / PKCE for Auth0/Okta/Keycloak, SAML XSW / signature stripping for
+Shibboleth/ADFS, GraphQL introspection / `node(id:)` IDOR for
+Apollo/Hasura, IIS `~1` shortname for Microsoft-IIS, Node prototype
+pollution for Express + lodash/qs, `__NEXT_DATA__` / source-map leaks for
+Next.js/Nuxt, rate-limit / response-substitution for `/verify` MFA flows.
+The first 3 HUNT actions must match the named stack — do not start a
+generic IDOR sweep when the recon evidence points at a specific class.
```

No new tools. No new files. Zero maintenance cost.

### Item 3 — cadence trigger labs (functional)

```
evidence/
└── cadence-labs/
    ├── README.md
    ├── 403_ladder.py   (Flask, port 5001)   → forces 3-bypass triad
    └── stall_target.py (Flask, port 5002)   → forces [STALL] auto-rotate
tests/
└── test_autopilot_cadence.py                → 5 pytest cases
```

`403_ladder.py` exposes one route per stack — Apache (trailing `;`), nginx
(off-by-slash), IIS (`~1` shortname), Node (X-Forwarded-For trust-proxy).
Each route 403s by default and 200s only when the stack-specific bypass is
applied. Forces the LLM to emit the actual 3-bypass triad instead of
treating the cadence block as a theoretical instruction.

`stall_target.py` returns a byte-identical 404 envelope to every probe.
Three consecutive identical responses to the same hypothesis must trigger
`[STALL] <diagnosis>` + `pick_next_lane`. No way for the LLM to
short-circuit; the only escape is rotation.

`tests/test_autopilot_cadence.py` is a **lexical** regression — it does
not run autopilot. It guarantees the 5 cadence headings, payload markers,
14 Question→Tool reference rows, Step 0.5/0.6 anchors, and 3 cadence lab
artifacts stay in tree.

```bash
$ python3 -m pytest tests/test_autopilot_cadence.py -v
... 5 passed
```

## Lessons from the design pivot

1. **Don't pre-package training-data recall** — if the LLM already knows
   OAuth `redirect_uri` bypass families, write a prompt line that names
   the class and forces explicit recall. Don't ship a markdown file that
   re-enumerates them.
2. **Maintenance cost compounds** — a regex selector + 8 markdown files
   + a README + a Contributing protocol is a permanent maintenance
   surface. A 12-line prompt block is none.
3. **The Cadence block already names per-stack payloads** — duplicating
   them in `hunt-memory/fingerprint_classes/*.md` creates two sources of
   truth that drift independently.
4. **The actual missing signal is "force the LLM to commit to a stack
   name"** — not "give the LLM more information about that stack." Step
   0.6 captures that distinction in 12 lines.
