# Supply-Side Form-Data Coverage Validation

**Date:** 2026-05-17
**Target:** DVWA v1.10 @ 127.0.0.1:8080 (security=low, admin/password)
**Goal:** Resolve framing error from `v4.4.1-smoke-test-summary.md` — was DVWA truly a "json_inject_probe blind spot"? **No.** Probe is JSON-only by design, but the *supply-side toolchain* (recon → playwright → scanner) is body-format-agnostic.

---

## Framing Error in Prior Summary

Earlier draft claimed:

> "All 8 DVWA findings were produced manually... json_inject_probe v4.4.1 only targets JSON bodies."

User correctly challenged this — the implication was misleading. **json_inject_probe is one tool among many.** The full supply chain covers form-data without any code change.

## Layer-by-Layer Coverage

| Layer | Body-format aware? | Captures form-data? | Evidence |
|---|---|---|---|
| `recon_engine.sh` (httpx/katana/gau/waymore) | URL-only | yes (URL + method) | live/httpx_full.txt |
| `playwright-cli` browser probe | full (headers+body+response) | **yes** | exec_post_capture.txt |
| Nuclei custom templates (`request.body:`) | full | yes | /tmp/nuclei_dvwa/sqli_form.yaml fired critical |
| `tools/json_inject_probe` v4.4.1 | **JSON-only by design** | no | not applicable to DVWA endpoints |

The probe is the **only** layer with a body-type restriction, and that restriction is intentional: it is a JSON-fuzzing primitive within a larger toolchain.
