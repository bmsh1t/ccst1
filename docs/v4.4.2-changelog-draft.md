```markdown
## v4.4.2 — Host:Port Lab Targets + Recon Exit-Code Stability (May 2026)

### Fixed
- `tools/target_paths.py::classify_target`: host:port targets (`127.0.0.1:3000`,
  `localhost:8080`, `app.example.com:8443`) previously raised
  `ValueError("invalid IP/CIDR target")` because the strict-digits regex matched
  but `ip_network()`/`ip_address()` both rejected the port form. The new
  host:port detection block runs before the strict-digits raise: numeric port
  1..65535, host is either a valid `ip_address` or a DNS-safe token.
- `tools/recon_engine.sh`: five sequential defects that mis-reported
  `Recon: Skipped` in `hunt.py`'s end-of-run dashboard even after a successful
  URL collection on ip/cidr/list targets.
  - Phase-1 host:port classification — routes `127.0.0.1:3000`-style inputs to
    `TARGET_KIND="ip"` instead of attempting `subfinder`/`assetfinder` against
    a non-domain.
  - Phase-4 gau/waymore skip — archive sources are domain-keyed; ip/cidr inputs
    either error out or hang on external lookups. Skip with a warn log instead.
  - Phase-5/7/Summary pre-seed — the ip/cidr branch skips wafw00f/origin/JS-secrets
    work, but later summary blocks still read those files via `< file`. Bash
    redirect-error noise on stderr was harmless but the missing files caused
    downstream `set -e` faults. Empty result files are now `touch`-ed up front.
  - Phase-8 GitHub-org dedupe pipeline — `grep -v '^$'` returns exit code 1
    on empty input, which under `set -euo pipefail` killed the entire recon
    script before the final summary printed. Appended `|| true` to the dedupe
    pipeline.
  - `cleanup_auth_tmpfiles` EXIT trap — the trap's final command was
    `[ -n "${WAFW00F_HEADERS_FILE:-}" ] && [ -f "$WAFW00F_HEADERS_FILE" ] && rm -f`,
    which evaluates to false (exit 1) when the headers file is unset. EXIT traps
    propagate their return code to the script's exit code, so a successful recon
    still returned 1 to `hunt.py`. Added explicit `return 0` at end of trap.
- `tools/vuln_scanner.sh` Check 6 (CVE nuclei lane): nuclei `-tags cve`
  against a target with hundreds of URLs can run for 90+ minutes, blocking
  every other scanner check. Added `timeout --preserve-status` wall-clock cap:
  `quick` mode 180s, `standard`/`full` mode 600s. Operators can override via
  `BB_CVE_TIMEOUT=<seconds>` env var. Switched from `cat <file> | nuclei` to
  `nuclei -l <file>` for cleaner timeout signalling.

### Tests
- `tests/test_hunt_target_types.py` (+4 cases): host:port classification —
  ipv4-with-port (`127.0.0.1:3000`, `192.168.1.10:8443`), domain-with-port
  (`localhost:8080`, `app.example.com:8443`), port-range rejection (port 0,
  port 99999).
- Full suite: 1450 passing locally (excluding network-dependent
  `tests/test_disclosure_search.py`).

### Validation
- `docs/v4.4.1-smoke-test-summary.md` — 13 manual findings (5 Juice Shop +
  8 DVWA) catalogued with attack-path deduction.
- `docs/v4.4.1-supply-side-coverage.md` — supply-side toolchain coverage
  matrix proving the json_inject_probe JSON-only restriction is by-design,
  not a form-data blind spot. The browser-probe + custom-template + scanner
  layers cover form-data.
- End-to-end verification:
  - `bash tools/recon_engine.sh 127.0.0.1:3000 --quick` → exit 0
  - `bash tools/recon_engine.sh 127.0.0.1:8080 --quick` → exit 0
  - `python3 tools/hunt.py --target 127.0.0.1:3000 --quick --scanner-skip cves`
    → `Recon: Done | Scan: Done`
  - `python3 tools/hunt.py --target 127.0.0.1:8080 --quick` (incl. CVE lane)
    → `Recon: Done | Scan: Done` in 544s with CVE phase capped at 180s

### Auto-discoveries (bonus, not in manual sweep)
- Juice Shop: 7 additional `auth_bypass` findings on
  `/api/Challenges`, `/api/Feedbacks`, `/api/Hints`, `/api/Products`,
  `/api/Quantitys`, `/api/SecurityQuestions` (all true-positive — Juice Shop
  exposes these unauthenticated).
- DVWA: critical `dvwa-default-login` (admin/password); medium
  `configuration-listing` on `/config/`.
- 3 medium exposure findings on `/metrics` (Prometheus), `/robots.txt`,
  `/.well-known/csaf/provider-metadata.json`.
```
